Cleaning Nashville Housing Data

Practicing Data Cleaning

View code on GitHub

Summary:

This project focuses on cleaning a dataset for future use. The goals of the project were to change date format, populate any missing data values, separate addresses, convert any inconsistant values like shorthand vs longhand naming convension, remove duplicates, and to delete excess columns from a database copy. I followed Alex the Analyst's YouTube video to complete this project.

Data Source:

Alex The Analyst's GitHub page (September 2023).

Results

The data was cleaned by:

  1. changing the date format with CAST,
  2. populating missing data values in PropertyAddress field,
  3. separating the PropertyAddress and OwnerAddress fields using SPLIT()[OFFSET()],
  4. converting shorthand Y/N to longhand Yes/No using CASE-WHEN,
  5. removing duplicates using a CTE.

Step 2: Standardize the date format

Change the sales date field from the datetime format to date.

  1. Change the SaleDate field data type from "datetime" to "date" with CAST().
×

Step 3: Fix missing data

Populate null property addresses where there is a related ParcelID.

  1. Check property address values for nulls.
  2. The ParcelID is associated with the PropertyAddress.
  3. Populate PropertyAddress by using ParcelID.
  4. Use a JOIN on ParcelID to find and replace PropertyAddress if null.
  5. Update Nashville table with new values.
×

Step 4: Split address

Split the property address and owner address into the street address, city, and state.

  1. Split the PropertyAddress values into Address and City using SPLIT() at the "," delimiter.
  2. Split the OwnerAddress values into Address, City, and State using SPLIT() and offset at the "," delimiter.
  3. Update the Nashville table with the new columns.
×

Step 5: Fix inconsitant values

Change inconsitant Y/N values to Yes/No in the SoldAsVacant field.

  1. Check SoldAsVacant column value range to identify inconsistancies: Y, N, Yes, No values.
  2. Change the Y/N values to Yes/No using WHEN THEN ELSE statements.
  3. Update the Nashville table with the new values in the Result column.
×

Step 6: Remove Duplicates and Unused Columns

Delete duplicates and unused columns in the copied database.

  1. Find duplicate values by partitiong data using a CTE.
  2. Check that the duplicates are gone with the CTE.
  3. Save the table containing no duplicates.
  4. Drop unused columns for data visualization and save as a new table.
  5. Save queries to GitHub.
