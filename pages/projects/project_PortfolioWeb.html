Step 1: Planning and Content Selection

I began by planning the website's structure, selecting the most relevant content, and defining the overall aesthetic.

  1. Research necessary content for a project portfolio website.
  2. Gather and review my most recent project files.
  3. Create a basic wireframe in Figma for the Home, About, and Projects pages.
  4. Determine the overall flow of the website.
×

Step 2: Design and Branding

I ensured that my personal brand identity was reflected throughout the website by crafting a visually appealing and consistent design.

  1. Determine the color palette for the website.
  2. Choose the fonts for the website.
  3. Determine the appropriate font sizes and styles for the text on the website.
  4. Choose if objects will have rounded corners or not.
×

Step 3: Development

Leveraging web development skills, I built the website using HTML, SCSS, and JavaScript for added interactivity.

  1. Open a new VS Code Workspace and create the necessary folders and page files.
  2. Open the index file and create the Home page HTML. Open the other website page files and write their HTML with placeholder images and text.
  3. Create the styles sheets in SCSS. I started with colors, fonts, and base page styles.
  4. Create styles sheets for each webpage.
  5. Add JavaScript to open and close modals.
×

Step 4: Content Creation

I curated and created content for each section, including project descriptions, a bio, and contact information.

  1. Determine what images I needed for each page.
  2. Created the text for each section on the pages, starting with the home page.
  3. Added the contact links.
  4. Review the projects and write a summary, steps taken, and results sections for each.
×

Step 5: Testing and Optimization

Rigorous testing and optimization were conducted to guarantee a seamless user experience across various devices and browsers.

  1. Find the screen sizes where each part of the design breaks.
  2. Determine the correct display, margin, padding attributes for the affected design part.
  3. Implement media queries for each screen size.
  4. Check that the pages' designs don't break at small and large screen sizes.
  5. Ensure each link goes to the corresponding page.
×

Step 6: Deployment

Once the website was polished and ready, it was deployed using the GitHub Pages hosting platform, making it accessible to visitors.

  1. Add webpages to GitHub repo.
  2. Create a GitHub page.
